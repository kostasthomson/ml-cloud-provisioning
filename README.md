# ML Cloud Provisioning - Energy-Aware Resource Allocation

A machine learning system for energy-aware cloud resource allocation. Trains neural networks on simulation data from the CloudLightning Simulator to predict optimal hardware type selection that minimizes energy consumption.

## Overview

This project implements an AIaaS (AI as a Service) solution for cloud provisioning decisions. It learns from CSV logs generated by the C++ CloudLightning Simulator to make energy-efficient hardware type selections for incoming tasks.

**Key Goals:**
- Minimize energy consumption in cloud resource allocation
- Architecture-agnostic model (works with any number of cells)
- Cell-level logging for distributed training data
- Real-time inference via REST API

## Features

### Current CSV Features (25 columns from C++ Simulator)

**Task Features:**
- `num_vms`: Number of VMs requested
- `cpu_req`: vCPUs per VM
- `mem_req`: Memory per VM (GB)

**Cell State Features:**
- `util_cpu_before`: CPU utilization before allocation
- `util_mem_before`: Memory utilization before allocation
- `avail_cpu_before`: Available CPUs
- `avail_mem_before`: Available memory (GB)
- `avail_storage_before`: Available storage (TB)
- `avail_accelerators_before`: Available accelerators

**Total Capacity:**
- `total_cpu`: Total CPUs in hardware type
- `total_mem`: Total memory (GB)
- `total_storage`: Total storage (TB)
- `total_accelerators`: Total accelerators

**Network:**
- `avail_network`: Available network bandwidth
- `total_network`: Total network bandwidth
- `util_network`: Network utilization

**Power Model:**
- `cpu_idle_power`: CPU idle power (W/core)
- `cpu_max_power`: CPU max power (W/core)
- `acc_idle_power`: Accelerator idle power (W)
- `acc_max_power`: Accelerator max power (W)

**Compute Capability:**
- `compute_cap_per_cpu`: FLOPS per CPU core
- `compute_cap_acc`: FLOPS per accelerator

**Metadata:**
- `energy_kwh`: Estimated energy consumption
- `chosen_hw_type`: Selected hardware type (label, -1 if rejected)
- `accepted`: Task acceptance status (1/0)

### Hardware Types
1. **CPU**: Standard CPU-only servers
2. **GPU**: GPU-accelerated servers
3. **DFE**: Dataflow Engine (FPGA) servers
4. **MIC**: Many Integrated Core (Xeon Phi) servers

## Usage

### Training Pipeline

1. **Generate training data** (run C++ simulator first):
   ```bash

   ./cl_sim ../input/CellData.json ../input/BrokerData.json ../input/AppData.json
   ```

2. **Merge cell CSVs and train**:
   ```bash
   cd scripts
   python train_pipeline.py
   ```

3. **Or train directly**:
   ```bash

   python train_agent.py ../training/training_data.csv
   ```

## Model Architecture

### EnergyAwareNN
```
Input (features) → Linear(64) → ReLU → Dropout(0.3)
                → Linear(64) → ReLU → Dropout(0.3)
                → Linear(32) → ReLU → Dropout(0.2)
                → Linear(4)  → HW Type Prediction
```

### Training
- **Loss**: CrossEntropyLoss
- **Optimizer**: Adam (lr=0.001)
- **Normalization**: StandardScaler
- **Split**: 70% train, 15% validation, 15% test

## Integration with C++ Simulator

The simulator outputs per-cell decision logs to:
```
cloudlightning-simulator/output/improved/cell_{id}_decisions.csv
```

The Python pipeline:
1. Copies CSV files to `training/simulation_runs/`
2. Merges all cells into `training/training_data.csv`
3. Trains the model on combined data
4. Saves model to `models/energy_aware/nn/model.pth`

## Energy Optimization

The model learns to minimize energy consumption by:
1. Understanding power characteristics of each hardware type
2. Considering current resource utilization
3. Matching task requirements to efficient hardware
4. Balancing load across available resources

Energy is calculated as:
```
E = (P_idle + util × (P_max - P_idle)) × duration / 3600000 [kWh]
```
